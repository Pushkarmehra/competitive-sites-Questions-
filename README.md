### Problem #1: Two Sum

|----------|----------------|------------------|-------------|
| Brute Force | O(n¬≤) | O(1) | Nested loops to check all pairs |

### Problem #5: Longest Palindromic Substring
| Approach | Time Complexity | Space Complexity | Description |
|----------|----------------|------------------|-------------|
| Brute Force | O(n¬≥) | O(1) | Check all substrings |
| Expand Around Centers | O(n¬≤) | O(1) | Expand around each possible center |
| Dynamic Programming | O(n¬≤) | O(n¬≤) | Bottom-up DP approach |

**Recommended:** Expand Around Centers for optimal space complexity

### Problem #9: Palindrome Number
| Approach | Time Complexity | Space Complexity | Description |
|----------|----------------|------------------|-------------|
| String Conversion | O(log n) | O(log n) | Convert to string and compare |
| Integer Reversal | O(log n) | O(1) | Reverse integer and compare with original |

**Recommended:** Integer Reversal approach for optimal space complexity

### Problem #48: Rotate Image
| Approach | Time Complexity | Space Complexity | Description |
|----------|----------------|------------------|-------------|
| Extra Matrix | O(n¬≤) | O(n¬≤) | Copy to new matrix |
| In-place Rotation | O(n¬≤) | O(1) | Transpose then reverse rows |

**Recommended:** In-place rotation for optimal space complexity

### Problem #189: Rotate Array
| Approach | Time Complexity | Space Complexity | Description |
|----------|----------------|------------------|-------------|
| Extra Array | O(n) | O(n) | Copy to new array |

### Problem #1957: Delete Characters to Make Fancy String

### Problem #3487: Maximum Unique Subarray Sum After Deletion
| Approach | Time Complexity | Space Complexity | Description |
|----------|----------------|------------------|-------------|

**Recommended:** Hash Array approach for optimal performance with fixed constraints

## üìù Solution Highlights

### Problem #48: Rotate Image (In-place Matrix Rotation)

**Key Insight:** Rotate matrix 90¬∞ clockwise by first transposing then reversing each row.

**Algorithm:**
1. Transpose the matrix (swap matrix[i][j] with matrix[j][i])
2. Reverse each row

**Time:** O(n¬≤), **Space:** O(1)

### Problem #898: Bitwise ORs of Subarrays

**Key Insight:** Use the property that OR operations can only increase or maintain bits, leading to at most 32 unique values per position.

**Algorithm:**
1. For each element, compute OR with all previous subarrays ending at current position
2. Use set to track unique OR values

**Time:** O(n log(max_or)), **Space:** O(unique_ors)

### Problem #3487: Maximum Unique Subarray Sum After Deletion

**Key Insight:** Since we can delete any elements, we want to keep only positive unique elements. If no positive elements exist, we keep the maximum element.

**Algorithm:**
1. Use a fixed-size hash array (size 201) to handle the constraint -100 ‚â§ nums[i] ‚â§ 100
2. First pass: Mark unique elements and sum positive ones
3. If sum is 0 (no positive elements), find the maximum unique element

**Edge Cases:**
- All elements are negative or zero
- All elements are the same
- Mix of positive and negative duplicates
